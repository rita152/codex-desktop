# Docs (docs/)

## Overview
Short “current implementation” docs for end-user visible features.

## Conventions
- Title should clearly state it describes the **current implementation**.
- Include:
  - prerequisites / limitations
  - a concrete workflow (steps)
  - troubleshooting section
  - implementation pointers (file paths)

## Files
- `git.md`: current Git integration behavior + implementation pointers
- `remote-servers.md`: SSH remote servers + `remote://` rules + troubleshooting
- `settings.md`: settings UI + persistence behavior + implementation pointers
